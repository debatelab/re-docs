Ensemble Generation with the module re_model.ensemble_generation¶

The methods in this module can be used to generate an ensemble of randomly generated runs of reflective equilibrium processes. The generated data will be saved in a csv-file, with the following columns (all complex data-types are stored as strings and have to be parsed if needed):

FEATURES OF THE DIALECTICAL STRUCTURE

  • model_name: A name of the model as string.

  • ds: The dialectical structure as list of int-lists. The first numbers of each list represent the premises, the last one the conclusion.

  • n_sentence_pool: Number of unnegated sentences (half the full size).

  • ds_arg_size: Number of arguments.

  • ds_infer_dens: The inferential density of the structure.

  • ds_n_consistent_complete_positions: Number of dialectically complete and consistent positions.

  • ds_mean_prem: Mean number of premises per argument.

  • ds_variance_prem: Variance of the number of premises per argument.

  • tau_truths: Propositions that are true in every complete consistent position.

  • principles: A list of tuples of the form (principle, multiplicity) with multiplicity indicating the multiplicity of the principle. A sentence counts as a principle iff it occurs in at least one argument as premise and it or its negation does not occur as a conclusion in an argument. The multiplicity counts in how many argument the sentence occurs as premise.

PROCESS-FEATURES

(Sentences \(\mathcal{C}_a\) are an axiomatic basis of \(\mathcal{C}_1\) (with sentences from \(\mathcal{C}_2\) only) iff \(\mathcal{C}_a\) dialectically implies \(\mathcal{C}_1\) and there is no proper subset of \(\mathcal{C}_a\) such that it implies \(\mathcal{C}_1\) (and \(\mathcal{C}_a \subset \mathcal{C}_2\)).)

  • init_final_coms_simple_hamming: Simple hamming distance between initial and final commitments.

A function to generate an ensemble of RE-processes.

The ensemble generation proceeds along the following steps:

  1. param_dia_structures should provide a fixed sequence of tupels in the form <n_arguments_min, n_arguments_max, n_sentences>.

  2. For each tupel <n_arguments_min, n_arguments_max, n_sentence_pool> the function randomly generates a number of dialectical structures (as defined by n_dialectical_structure) with a number of arguments between n_arguments_min and n_arguments_max and a sentence-pool of size n_sentence_pool (without negations).

  3. For each generated dialectical structure the function then generates a set of initial commitment-sets (not necessarily the same for each dialectical structure).

  4. For each such dialectical-structure/initial-commitments pair, the function runs RE-processes for each model and each specified set of parameters for the achievement function.

    • The result of each run is saved as a row in the table.

    • If save_branches == True each alternative branch of the re-process is saved as an additional row. (Fixed points of RE-processes are not unique, since a process can be underdetermined due to different available choices the process might have at some steps.)

Parameters

  • output_file – File where the date will be saved.

  • param_dia_structures (List[Tuple[int, int, int]]) – List of tupels of the form <n_arguments_min, n_arguments_max, n_sentence_pool>.

  • n_dialectical_structure (int) – Number of dialectical structure (per tupel and model).

  • n_premises_max (int) – Maximum number of premises in an argument.

  • n_premises_fixed (bool) – If True all arguments will have exactly n_premises_max premises.

  • n_initial_commitments (int) – The number of initial commitments.

  • alpha_resolution (Optional[int]) – If not None it will be used to generate a spectrum of weight for the achievement function. Resolution of 10 means to generate a spectrum of 0.1,0.2,…,0.9 for each weight.

  • alpha_values (Optional[List[List[int]]]) – List of weights (\(\alpha\)-values) for the achievement-function (order: account, sytematicity, faithfulness).

  • n_principles (int) – Number of principles to be guaranteed by the algorithm generating the dialectical structures.

  • re_implementations (List[Tuple[str, str]]) – String-representation of the models to be used (defined by module- and class name) (Must be instantiations of remodeldescription.basics.ReflectiveEquilibrium).

  • save_all_branches (bool) – If True every possible branch of an re-process is saved as an individual row in the table.

  • save_preliminary_results (bool) – If True the data-generation process will save temporary file every other amount of rows as defined by preliminary_results_interval.

  • preliminary_results_interval (int) –
